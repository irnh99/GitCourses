Git Commands

  • config: This command sets the author name and email address respectively to be used with your commits.
  • init: This command is used to start a new repository.
  • clone: This command is used to obtain a repository from an existing URL.
  • add: This command adds a file to the staging area.
  • commit: This command records or snapshots the file permanently in the version history.
  • diff: This command shows the file differences which are not yet staged.
  • reset: This command unstages the file, but it preserves the file contents.
  • status: This command lists all the files that have to be committed.
  • rm: This command deletes the file from your working directory and stages the deletion.
  • log: This command is used to list the version history for the current branch.
  • show: This command shows the metadata and content changes of the specified commit.
  • tag: This command is used to give tags to the specified commit.
  • branch: This command lists all the local branches in the current repository.
  • checkout: This command is used to switch from one branch to another.
  • merge: This command merges the specified branchâ€™s history into the current branch.
  • remote: This command is used to connect your local repository to the remote server.
  • push: This command sends the committed changes of master branch to your remote repository.
  • pull: This command fetches and merges changes on the remote server to your working directory.
  • stash: This command temporarily stores all the modified tracked files.
